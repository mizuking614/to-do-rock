# 統計・分析画面 (stats_view.dart)

## 概要

ホーム画面のナビゲーションから遷移する画面。  
Rockの順番攻略状況を軸に、生産性の傾向をグラフで可視化する。  
「Rockを先に倒せているか」が主要な指標。

---

## コアコンセプト：Rock攻略率が主指標

- **Rockを何番まで攻略したか** = 今日・今週の達成度の基準
- PebbleとSandは補助的な指標として表示

---

## 表示要素

### 1. ヘッダー
- 「統計」タイトル
- 期間切替タブ（「今日」「今週」「今月」「全期間」）

### 2. Rockサマリーカード（最上部・強調表示）
- 現在のRock攻略番号（例: "Rock #3 まで攻略済み"）
- 攻略率プログレスバー（完了Rock数 / 全Rock数）
- 連続攻略日数（ストリーク: 毎日1つ以上Rockを完了した日数）

### 3. カテゴリ別完了数カード（横並び3枚）
| カード | 表示内容 |
|---|---|
| 🔒 Rock | 完了数 / 全Rock数 |
| 🪨 Pebble | 完了数 / 全Pebble数 |
| 🏖️ Sand | 完了数 / 全Sand数 |

### 4. 週別Rockチャート
- 棒グラフで各曜日に完了したRockの番号進捗を表示
- Rock完了日はアクセントカラーでハイライト
- Rockを一度も完了しなかった日はグレー

### 5. Pebble/Sand達成率
- 横棒グラフでそれぞれの完了率を表示

### 6. アドバイスコメント
- 「今週はRockを毎日攻略できています！」
- 「Sandばかり完了していてRockが進んでいません」等の自動生成コメント

### 7. 完了済みタスク一覧（アーカイブ）
- カテゴリ・完了日時順のリスト
- 「もっと見る」で展開

---

## 状態管理（stats_view_model.dart）

| 状態 | 型 | 説明 |
|---|---|---|
| `selectedPeriod` | StatsPeriod | 集計期間 |
| `rockProgress` | int | 攻略済みRock数 |
| `totalRocks` | int | 全Rock数 |
| `rockStreak` | int | 連続Rock攻略日数 |
| `categoryStats` | Map\<Category, Stat\> | カテゴリ別完了数 |
| `dailyRockProgress` | List\<int\> | 曜日別Rock完了数 |
| `isLoading` | bool | データ集計中 |

---

## ViewModel の責務

1. `TaskRepository` から期間内のタスクを取得
2. Rock・Pebble・Sandごとに完了数を集計
3. Rock連続攻略日数（ストリーク）を算出
4. 期間切替時に再集計してUIを更新
5. 傾向からアドバイスコメントを生成

---

## 使用するRepository

| 依存先 | 用途 |
|---|---|
| `TaskRepository` | 完了タスクの取得・集計 |

---

## エラー・ローディングの扱い

- 集計中: ローディングスピナー
- データなし: 「まだ記録がありません」メッセージ
