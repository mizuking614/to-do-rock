# ホーム画面 (home_view.dart)

## 概要

アプリ起動後にユーザーが最初に見るメイン画面。  
タスクは **Rock（順番ロック制）・Pebble（自由）・Sand（自由）** の3種類に分類される。  
Rockは1番から順に完了しないと次に進めない。PebbleとSandは自由に完了できる。

---

## コアコンセプト：Rock のロック制

- Rockタスクは **登録した順番（1, 2, 3...）に必ずこなす必要がある**
- 現在の番号のタスクだけが操作可能で、それ以降は 🔒 ロック状態
- 完了すると自動的に次の番号がアンロックされる

---

## 表示要素

### 1. ヘッダーエリア
- アプリ名（"to do rock"）ロゴ・タイトル
- タスク追加ボタン（右上の「＋」アイコン）
- 検索ボタン（虫眼鏡アイコン）

### 2. 進捗サマリーエリア
- Rockの現在進行番号（例: "Rock 2 / 5"）
- Rockのプログレスバー
- Pebble・Sandそれぞれの残件数バッジ

### 3. フィルタータブ（横スクロール）
- 「すべて」「🔒 Rock」「🪨 Pebble」「🏖️ Sand」「完了済み」

### 4. Rockセクション（ロック制リスト）

#### アクティブタスク（現在の番号）
- 番号バッジ（例: **#1**）＋ タスクタイトル（大きめ・強調）
- 完了チェックボックス（タップ可能）
- 期限・メモ有無アイコン

#### ロック中タスク（次以降の番号）
- 番号バッジ＋ タスクタイトル（グレーアウト）
- 🔒 ロックアイコン
- チェックボックスは非活性（タップ不可）

### 5. Pebbleセクション（自由リスト）
- チェックボックス・タイトル・期限（任意）
- どの順番でも完了可能
- 長押しで並び替え可

### 6. Sandセクション（自由リスト）
- Pebbleと同じUI
- 表示優先度は最下部

### 7. タスク追加FAB
- 右下固定の「＋」ボタン
- タップでタスク作成画面へ（カテゴリ選択から開始）

---

## インタラクション

| 操作 | 動作 |
|---|---|
| Rockのアクティブタスクを完了 | アニメーションで完了→次の番号がアンロック |
| Rockのロック中タスクをタップ | 「前のタスクを完了してから進めます」トースト表示 |
| Pebble/Sandのタスクをタップ | タスク詳細画面へ遷移 |
| タスクを左スワイプ | 削除ボタンを表示 |
| タスクを右スワイプ（Pebble/Sand） | Rockに昇格 / カテゴリ変更ショートカット |
| Rockのタスクを長押し | 順番の並び替えモードへ |

---

## 状態管理（home_view_model.dart）

| 状態 | 型 | 説明 |
|---|---|---|
| `rockTasks` | List\<Task\> | Rockタスク（順番付き） |
| `pebbleTasks` | List\<Task\> | Pebbleタスク |
| `sandTasks` | List\<Task\> | Sandタスク |
| `currentRockIndex` | int | 現在アクティブなRockの番号（0始まり） |
| `selectedFilter` | FilterType | 選択中のフィルター |
| `isLoading` | bool | データ取得中フラグ |

---

## ViewModel の責務

1. 起動時に `TaskRepository` から全タスクを取得しカテゴリ別に分類
2. `currentRockIndex` を管理し、アクティブなRockタスクを特定
3. Rockタスク完了時に `currentRockIndex` をインクリメント
4. Pebble/Sandのチェックは独立して処理

---

## 使用するRepository

| 依存先 | 用途 |
|---|---|
| `TaskRepository` | タスクのCRUD・順番管理 |

---

## エラー・ローディングの扱い

- データ取得中: ローディングスピナー
- ロック中タスクをタップ: 「前のタスクを完了してください」トースト
- 削除後: 「削除しました」Snackbar＋アンドゥ
