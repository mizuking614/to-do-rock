# カレンダービュー画面 (calendar_view.dart)

## 概要

ホーム画面のナビゲーションから遷移する画面。  
期限が設定されたタスクをカレンダー形式で管理する。  
RockタスクはロックアイコンつきのドットでPebble/Sandと区別して表示。

---

## コアコンセプト：期限とRockロック状態の可視化

- Rockタスクは 🔒 マークつきで表示（ロック状態が一目でわかる）
- ロック中のRockタスクは詳細から完了不可
- 期限切れのRockは特に赤でアラート表示（ロックが解除されていないと進めない）

---

## 表示要素

### 1. ヘッダー
- 「カレンダー」タイトル
- 「今日」ボタン（現在日付に戻る）
- 月移動ボタン（「‹」「›」）

### 2. カレンダーエリア（月表示）
- 各日付セルの表示内容:
  - タスクがある日: カテゴリ別のドット（最大3つ）
  - Rockの期限日: 🔒 マークつきドット
  - 今日: 丸ハイライト
  - 選択日: 枠線ハイライト
  - 期限切れRockがある日: 赤いドット
- カレンダー上部に曜日ヘッダー（日〜土）
- 左右スワイプで月移動

### 3. 凡例
- 🔒 Rock（ロック制）/ 🪨 Pebble / 🏖️ Sand / 🔴 期限切れ

### 4. 選択日のタスクリスト
- Rockタスク（ロック状態を表示）
- Pebbleタスク
- Sandタスク
- 「タスクがない場合」: 「この日にタスクはありません」メッセージ
- 「＋ タスクを追加」リンク

#### タスクカードの表示
- 左端にカテゴリカラーのアクセントライン
- タイトル・カテゴリバッジ
- Rockロック中の場合: 🔒アイコン＋グレーアウト

---

## インタラクション

| 操作 | 動作 |
|---|---|
| カレンダーの日付をタップ | その日のタスクリストを表示 |
| 月移動ボタン / スワイプ | 前月・翌月へ切り替え |
| 「今日」ボタン | 今日の日付へ移動 |
| タスクカードをタップ | タスク詳細画面へ遷移（Rockロック中は閲覧専用） |
| 「＋ タスクを追加」 | タスク作成画面（その日を期限にプリセット） |

---

## 状態管理（calendar_view_model.dart）

| 状態 | 型 | 説明 |
|---|---|---|
| `focusedMonth` | DateTime | 表示中の月 |
| `selectedDate` | DateTime | 選択中の日付 |
| `tasksByDate` | Map\<DateTime, List\<Task\>\> | 日付ごとのタスクマップ |
| `currentRockIndex` | int | 現在アクティブなRock番号 |
| `isLoading` | bool | データ取得中フラグ |

---

## ViewModel の責務

1. 月切替時に `TaskRepository` から期限付きタスクを取得
2. 日付ごとに分類して `tasksByDate` を生成
3. Rock のロック状態（`currentRockIndex`）を参照してカード表示を制御
4. 選択日変更時にフィルタリングしてUIを更新

---

## 使用するRepository

| 依存先 | 用途 |
|---|---|
| `TaskRepository` | 期限付きタスク取得・Rock順番状態の参照 |

---

## エラー・ローディングの扱い

- データ取得中: リストエリアにローディングスピナー
- データ取得失敗: エラーメッセージ＋リトライ
- Rockロック中タスクをタップ: 詳細画面に遷移するが完了不可（ロックバナー表示）
