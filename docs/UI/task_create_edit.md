# タスク作成・編集画面 (task_create_edit_view.dart)

## 概要

FAB（＋ボタン）またはタスクカードのタップから遷移する画面。  
新規作成・既存編集を同一UIで行う。  
最初にカテゴリ（Rock / Pebble / Sand）を選ぶことで入力項目が変わる。

---

## コアコンセプト

- **Rock を選んだ場合**: Rockリストの末尾に追加される（順番は後から並び替え可能）
- **Pebble / Sandを選んだ場合**: 通常のタスクとして追加

---

## 表示要素

### 1. ヘッダー
- 「新しいタスク」または「タスクを編集」タイトル
- キャンセルボタン（左上）
- 保存ボタン（右上・タイトル入力後に活性）

### 2. カテゴリ選択エリア（最上部）
- 3択のセグメントボタン
  - 🔒 Rock（ロック制・順番必須）
  - 🪨 Pebble（自由）
  - 🏖️ Sand（自由）
- 選択したカテゴリに応じてUI下部が変化
- Rockを選んだ場合: 「Rockリストの末尾（#N）に追加されます」の案内テキスト表示

### 3. タイトル入力
- プレースホルダー: 「タスクのタイトルを入力」
- 最大100文字・画面表示時に自動フォーカス

### 4. 期限設定（Rock / Pebble のみ推奨表示）
- 「期限を設定」トグル
- ON時: 日付ピッカー表示
- Sandは期限フィールドを折りたたんで表示（任意感を強調）

### 5. メモ入力
- 複数行テキストエリア・最大500文字
- 全カテゴリ共通

### 6. サブタスク（Rock / Pebble のみ表示）
- サブタスクリスト＋「＋ サブタスクを追加」ボタン
- Sandには非表示（シンプルさを維持）

### 7. Rockの順番設定（Rockかつ編集モードのみ）
- 「Rockリスト内の順番」を数値入力で変更可能
- 現在のロック状態（アクティブ/ロック中）も表示

### 8. 削除ボタン（編集モードのみ）
- 画面下部「このタスクを削除」（赤テキストボタン）
- 確認ダイアログあり

---

## インタラクション

| 操作 | 動作 |
|---|---|
| カテゴリ切替 | アニメーションでUIが切り替わる |
| 保存 | バリデーション後に保存・前画面へ戻る |
| キャンセル（変更あり） | 「変更を破棄しますか？」確認ダイアログ |
| Rockを選択 | 追加先番号（#N）を自動計算して表示 |

---

## 状態管理（task_create_edit_view_model.dart）

| 状態 | 型 | 説明 |
|---|---|---|
| `isEditMode` | bool | 編集 or 新規作成 |
| `category` | Category | Rock / Pebble / Sand |
| `title` | String | タイトル |
| `dueDate` | DateTime? | 期限 |
| `memo` | String | メモ |
| `subTasks` | List\<SubTask\> | サブタスク（Rock/Pebbleのみ） |
| `rockPosition` | int? | Rockリスト内の位置（編集時） |
| `isSaving` | bool | 保存中フラグ |

---

## 使用するRepository

| 依存先 | 用途 |
|---|---|
| `TaskRepository` | タスクの保存・更新・削除・Rock順番管理 |

---

## エラー・ローディングの扱い

- タイトル未入力: 保存ボタン非活性
- 保存失敗: エラースナックバー
- Rockを削除する場合: 「後続タスクの番号が繰り上がります」旨の確認ダイアログ
